# 意图触发

不设 `/opc` 入口命令。自然语言就是入口。orchestrator 自动判断用户意图：

```
用户: 实现用户认证系统      → 识别为任务，启动管线
用户: 修复登录页样式错乱    → 识别为任务，启动管线
用户: 这个变量命名怎么样    → 识别为问答，直接回答
用户: 今天天气怎么样        → 静默，不介入
```

## 置信度与纠错

task-classifier 对每条意图输出置信度（0-1）。低于阈值时，主动向用户确认：

```
用户: 这个函数的性能怎么样
分类器: task(0.45) / question(0.55)  → 置信度低 → 追问: "需要我启动性能优化流程，还是先帮你分析？"
用户: 先分析一下               → 识别为问答，不启动管线
```

## 纠错指令

用户可以用简短的自然语言取消错误的管线启动：

| 纠错指令 | 效果 |
|----------|------|
| "不用启动管线" / "just answer" | 取消管线，作为普通问答处理 |
| "先不做了" / "cancel" | 取消管线，标记为 aborted |
| "这不是任务" / "not a task" | 取消并学习：将当前消息标记为问答样本 |

## 显式声明（补充信号）

用户可以在消息前加标记增强意图识别：

| 前缀 | 效果 |
|------|------|
| `!task <描述>` | 强制作为任务执行 |
| `? <问题>` | 强制作为问答处理 |

## 精确命令

保留三个精确命令用于手动控制：

| 命令 | 用途 |
|------|------|
| `/opc-status` | 查看管线进度 + 下一步建议 |
| `/opc-phase` | 手动跳转/重试某个阶段 |
| `/opc-nodes` | 查看当前阶段的节点选项 |
