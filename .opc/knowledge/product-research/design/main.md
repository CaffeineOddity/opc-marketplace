## 交互与界面（如适用）

### 用户交互模式

product-research 系统主要通过命令行界面（CLI）和 Agent 对话进行交互，不涉及传统 GUI 界面设计。

#### Skill 命令交互

```
/research <主题>        # 市场调研
/requirement <功能>     # 需求撰写
/brainstorm <挑战>      # 头脑风暴
/spec-driven-development # 规范驱动开发
```

#### Agent 调用交互

```shell
# 直接调用 Agent
Agent(subagent_type="product-kit:product-agent", prompt="...")

# 通过 /opc 命令自动调度
/opc <任务描述>
```

### 信息架构

#### Skill 层级架构
```
product-kit/
├── Skills（轻量级能力）
│   ├── /research      → 调研简报
│   ├── /requirement   → PRD
│   ├── /brainstorm    → 创意清单
│   └── /spec-driven-development → 规范文档
└── Agents（全面能力）
    ├── product-agent      → 全流程
    ├── startup-analyst    → 市场分析/财务建模
    └── business-analyst   → 业务分析/需求获取
```

#### 数据流转架构
```
用户输入
    ↓
/research → 调研简报
    ↓
/requirement → PRD
    ↓
/brainstorm → 创意清单（可选）
    ↓
/spec-driven-development → 规范文档
    ↓
交接 design-kit / dev-kit / growth-kit
```

## 边界条件

### 输入边界

| 输入类型 | 最小长度 | 最大长度 | 处理方式 |
|----------|----------|----------|----------|
| 调研主题 | 2 字符 | 500 字符 | 截断或拒绝 |
| 功能描述 | 10 字符 | 5000 字符 | 截断或拒绝 |
| 挑战描述 | 10 字符 | 2000 字符 | 截断或拒绝 |

### 输出边界

| 输出类型 | 最大长度 | 处理方式 |
|----------|----------|----------|
| 调研简报 | 10000 字符 | 分段输出 |
| PRD 文档 | 20000 字符 | 分段输出 |
| 创意清单 | 50 个创意 | 分批输出 |
| 规范文档 | 30000 字符 | 分段输出 |

### 处理边界

| 场景 | 超时时间 | 处理方式 |
|------|----------|----------|
| Skill 执行 | 120 秒 | 返回部分结果 + 提示继续 |
| Agent 调用 | 300 秒 | 返回部分结果 + 提示继续 |
| 数据获取 | 30 秒 | 返回缓存或错误提示 |

### 数据边界

| 数据类型 | 边界条件 | 处理方式 |
|----------|----------|----------|
| 用户故事 | 最多 50 个 | 建议拆分文档 |
| 功能需求 | 最多 100 个 | 建议拆分文档 |
| 创意数量 | 最多 100 个 | 建议分批头脑风暴 |
| 规范项 | 最多 200 个 | 建议拆分模块 |

## 异常与降级

### Skill 执行异常

| 异常类型 | 原因 | 降级策略 |
|----------|------|----------|
| 输入不明确 | 用户输入模糊或矛盾 | 引导用户澄清，提供示例 |
| 数据源不可用 | 外部数据源无法访问 | 使用缓存数据或标注不确定性 |
| 处理超时 | 任务复杂度过高 | 返回部分结果，提示继续 |
| 输出过长 | 结果超过长度限制 | 分段输出，提供摘要 |

### Agent 调用异常

| 异常类型 | 原因 | 降级策略 |
|----------|------|----------|
| Agent 不可用 | Agent 未安装或配置错误 | 回退到 Skill 执行 |
| 模型不可用 | 指定模型不可用 | 使用 fallback 模型 |
| 工具不可用 | 必需工具未授权 | 提示用户授权或使用替代方案 |
| 交接失败 | 下游系统不可用 | 保存输出，提示稍后交接 |

### 数据处理异常

| 异常类型 | 原因 | 降级策略 |
|----------|------|----------|
| 数据格式错误 | 输出格式不符合预期 | 自动修复格式 |
| 数据缺失 | 必需数据缺失 | 使用默认值或提示用户 |
| 数据冲突 | 数据之间存在矛盾 | 标记冲突，建议用户确认 |
| 版本冲突 | 文档版本不一致 | 提示用户合并或选择版本 |

### 降级模式

#### 完全降级
当核心功能不可用时：
1. 提示用户功能不可用
2. 提供替代方案（如手动操作指南）
3. 记录错误日志
4. 建议稍后重试

#### 部分降级
当部分功能不可用时：
1. 继续可用功能
2. 标记不可用功能
3. 提供替代方案
4. 建议稍后补充

#### 性能降级
当响应时间过长时：
1. 返回部分结果
2. 提供继续提示
3. 建议简化输入
4. 后台继续处理

## 可访问性 / 可用性（如适用）

### 命令可用性

#### 命令简洁性
- Skill 命令简短易记：`/research`, `/requirement`, `/brainstorm`, `/spec-driven-development`
- 支持命令别名和缩写
- 提供命令提示和自动补全

#### 错误提示友好性
- 错误信息清晰、具体
- 提供修复建议
- 避免技术术语

### 输出可用性

#### 结构化输出
- 使用 Markdown 格式
- 清晰的章节划分
- 表格和列表便于阅读

#### 可编辑性
- 输出可直接编辑
- 支持版本控制
- 易于复制和分享

### 文档可用性

#### 文档完整性
- 每个输出包含必要章节
- 提供模板和示例
- 标注必填和选填项

#### 文档可追溯性
- 标注数据来源
- 记录创建和更新时间
- 支持版本历史

### 学习曲线

#### 新用户引导
- 提供快速开始指南
- 内置示例和模板
- 渐进式功能介绍

#### 高级用户支持
- 支持自定义模板
- 提供高级参数
- 支持批量操作
