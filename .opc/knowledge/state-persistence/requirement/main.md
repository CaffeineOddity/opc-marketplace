# WHAT（要做什么）

OPC 状态持久化系统提供跨会话的状态管理能力，确保任务进度、决策和上下文在多个会话间保持一致。系统基于 MCP 工具实现，提供以下核心能力：

1. **会话管理** - 单窗口单任务模型，任务状态持久化
2. **检查点系统** - 创建、列出、回滚检查点
3. **Agent 交接** - 保留上下文的 Agent 间传递
4. **项目记忆** - 存储决策、模式、教训

**MCP 工具列表：**

| 工具 | 用途 |
|------|------|
| `opc_state_init` | 初始化新任务会话 |
| `opc_state_read` | 读取当前任务状态 |
| `opc_state_write` | 更新任务状态 |
| `opc_state_clear` | 清除当前任务 |
| `opc_checkpoint_create` | 创建检查点 |
| `opc_checkpoint_list` | 列出检查点 |
| `opc_checkpoint_rollback` | 回滚到检查点 |
| `opc_handoff` | 记录 Agent 交接 |
| `opc_memory` | 读写项目记忆 |
| `opc_sessions_list` | 列出所有会话 |

# WHY（为什么要做）

一人公司的工作经常被打断：

| 场景 | 问题 |
|------|------|
| 临时会议 | 任务进度丢失 |
| 下班休息 | 上下文遗忘 |
| 多日开发 | 决策难以追溯 |
| Bug 修复 | 需要回退到稳定状态 |

状态持久化系统解决：

| 痛点 | 解决方案 |
|------|----------|
| 进度丢失 | 任务状态持久化 |
| 上下文遗忘 | Agent 交接保留上下文 |
| 决策遗忘 | 项目记忆存储决策 |
| 回退困难 | 检查点支持回滚 |

## 功能性需求

### 核心功能

1. **会话管理**
   - 初始化：创建新任务会话，生成唯一 lock_id
   - 读取：查看当前任务进度、阶段状态、产物
   - 更新：记录阶段完成、产物文件、Agent 信息
   - 清除：放弃当前任务，释放窗口锁

2. **单窗口单任务模型**
   - 每个窗口只能有一个任务
   - lock_id 基于 PID + 时间戳
   - 防止多任务并行导致状态混乱

3. **检查点系统**
   - 创建：保存当前状态快照
   - 列出：显示所有可用检查点
   - 回滚：恢复到指定检查点
   - 用途：大改动前备份、失败后恢复

4. **Agent 交接**
   - 记录：保存 Agent 间传递的上下文
   - 内容：决策、约束、产物、问题
   - 格式：结构化交接文档

5. **项目记忆**
   - 分类：decision/pattern/lesson/constraint
   - 读写：支持读写项目级记忆
   - 持久化：存储在 `.opc/memory/project-memory.json`

### 辅助功能

1. **会话列表**
   - 显示所有活跃会话
   - 包含 lock_id、任务名、状态

2. **状态可视化**
   - 阶段图标：✅ 完成、🔄 执行中、⏳ 待执行、🚫 阻塞
   - 进度百分比
   - 产物列表

## 非功能性需求

- **性能**：
  - 状态读写 < 100ms
  - 检查点创建 < 500ms
  - 回滚操作 < 1秒

- **安全性**：
  - 状态文件存储在用户本地 `.opc/state/`
  - 不上传到远程服务器
  - Git 提交时自动排除

- **可靠性**：
  - 原子写入，避免损坏
  - 异常时保留当前状态
  - 检查点支持多次回滚

- **可用性**：
  - 自动管理，无需手动干预
  - 清晰的状态展示
  - 简单的回滚操作

## 不做什么（Non-goals）

1. **不支持多任务并行** - 一人公司专注更高效
2. **不支持远程同步** - 状态仅本地存储
3. **不支持加密** - 状态文件明文存储
4. **不支持版本控制** - 检查点独立管理
