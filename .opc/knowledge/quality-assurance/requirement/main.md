# WHAT（要做什么）

qa-kit 是一人公司质量保障系统的核心插件，提供完整的软件测试和质量保障能力，包括：

- **测试计划**：从需求生成结构化测试计划
- **缺陷报告**：标准化 Bug 报告格式
- **E2E 测试**：Playwright/Cypress 端到端测试
- **无障碍审计**：WCAG 2.2 合规性检查
- **Web 应用测试**：Playwright 测试工具包

# WHY（为什么要做）

一人公司开发者需要确保产品质量，但缺乏专业 QA 团队支持。qa-kit 通过专业化 Agent 和 Skills 提供：

1. **降低测试门槛**：自动化测试计划生成，无需专业 QA 背景
2. **提升测试效率**：E2E 测试自动生成，减少手工测试工作量
3. **保证合规性**：WCAG 无障碍审计，满足法规要求
4. **标准化流程**：缺陷报告结构化，便于追踪和修复

## 功能性需求

### FR-01: 测试计划生成
- 支持从需求文档自动生成测试计划
- 包含测试范围、测试用例、优先级排序
- 支持功能测试、集成测试、非功能测试分类

### FR-02: 缺陷报告生成
- 标准化缺陷报告格式
- 包含复现步骤、预期结果、实际结果
- 支持严重程度和优先级分类

### FR-03: E2E 测试生成
- 支持 Playwright 和 Cypress 框架
- 自动生成 Page Object Model
- 包含测试场景、断言、设置/清理

### FR-04: WCAG 无障碍审计
- 支持 WCAG 2.2 Level A/AA/AAA
- 自动化检测 + 手动验证指导
- 提供修复建议和代码示例

### FR-05: Web 应用测试工具包
- Playwright 测试自动化
- 页面交互、断言、最佳实践
- 测试生成和调试支持

## 非功能性需求

- 性能：
  - 测试计划生成 < 30s（中等复杂度需求）
  - WCAG 审计 < 60s（单页面）
  - E2E 测试生成 < 45s（标准场景）

- 安全性：
  - 测试数据脱敏
  - 敏感信息不记录
  - 测试环境隔离

- 可靠性：
  - 测试用例覆盖率 > 80%
  - 缺陷报告准确率 > 95%
  - WCAG 检测准确率 > 90%

- 可用性：
  - Skills 即插即用
  - 与 dev-kit 无缝集成
  - 测试报告自动生成

## 不做什么（Non-goals）

- 不执行测试（由 CI/CD 流水线执行）
- 不负责性能测试执行（由 ship-kit 监控）
- 不负责安全测试执行（由 security-auditor 负责）
- 不负责部署验证（由 devops-agent 负责）
