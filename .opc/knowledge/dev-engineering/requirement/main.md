# WHAT（要做什么）

dev-kit 是一人公司开发工程系统的核心插件，提供完整的软件开发能力覆盖，包括：

- **架构设计**：系统架构文档生成、技术选型决策
- **前端开发**：UI 实现、组件架构、性能优化
- **后端开发**：API 设计、数据层、服务架构
- **安全审计**：DevSecOps、OWASP 合规、漏洞评估
- **移动开发**：React Native/Flutter/原生开发
- **数据库架构**：数据建模、Schema 设计、迁移规划
- **性能工程**：性能分析、优化、基准测试
- **AI 工程**：AI 系统工程、模型部署、MLOps
- **提示词工程**：LLM 优化、提示词设计
- **技术文档**：API 文档、开发指南

# WHY（为什么要做）

一人公司开发者需要面对全栈开发的挑战，但精力有限。dev-kit 通过专业化 Agent 和 Skills 提供：

1. **降低认知负担**：每个 Agent 专注单一领域，提供专业指导
2. **提升开发效率**：自动化代码审查、架构设计、调试流程
3. **保证代码质量**：TDD 方法论、验证协议、安全审计
4. **加速知识传递**：技术文档自动生成、最佳实践内置

## 功能性需求

### FR-01: 架构设计能力
- 支持从需求生成架构设计文档
- 包含系统概览、组件分解、API 设计、数据模型
- 支持多种架构模式（单体、微服务、事件驱动）

### FR-02: 代码审查能力
- 四维度审查：Bug/安全/性能/可读性
- 自动检测常见漏洞和反模式
- 提供具体修复建议和代码示例

### FR-03: API 规范生成
- OpenAPI 3.1 规范自动生成
- 支持请求/响应 Schema 定义
- 包含认证授权模型

### FR-04: 前端开发支持
- 生产级前端界面生成
- shadcn/ui 组件集成
- Core Web Vitals 性能优化

### FR-05: 系统化调试
- 四阶段调试方法论
- 根因分析、模式匹配、假设验证
- 支持复杂问题定位

### FR-06: TDD 开发流程
- 红-绿-重构循环
- 测试优先开发方法论
- 完成验证协议

### FR-07: 图表生成
- 架构图、流程图、时序图
- 思维导图、时间线
- 专业 SVG 输出

### FR-08: MCP 服务器开发
- Python/TypeScript MCP 开发指南
- 工具定义和实现模式

## 非功能性需求

- 性能：
  - 代码审查响应时间 < 30s（1000 行代码）
  - 架构文档生成 < 60s（中等复杂度系统）
  - 图表生成 < 10s

- 安全性：
  - 代码审查覆盖 OWASP Top 10
  - 自动检测敏感信息泄露
  - 安全审计 Agent 使用 Opus 模型

- 可靠性：
  - 代码建议准确率 > 90%
  - 架构设计符合行业最佳实践
  - 调试方法论可复现

- 可用性：
  - Skills 即插即用
  - Agents 自动协作
  - 文档自动生成

## 不做什么（Non-goals）

- 不替代 IDE 的代码编辑功能
- 不提供 CI/CD 流水线执行（由 ship-kit 负责）
- 不负责产品需求分析（由 product-kit 负责）
- 不负责 UI/UX 设计（由 design-kit 负责）
- 不负责测试执行（由 qa-kit 负责）

- [ ] 所有 10 个 Skill 定义完整且可用
- [ ] auto-lint Hook 正常工作
- [ ] Agent 间协作流程清晰定义
- [ ] 文档完整覆盖所有功能
- [ ] 与其他 kit 的边界清晰
