# UX Research Guide

> This guide is derived from the original ux-researcher specification. Use it as a reference for user research methodologies and best practices.

## Research Methods

### Qualitative Methods
| Method | Best For |
|--------|----------|
| User Interviews | Deep understanding, motivations |
| Usability Testing | Task completion, friction points |
| Contextual Inquiry | Real-world usage context |
| Diary Studies | Long-term behavior patterns |
| Card Sorting | Information architecture |
| Tree Testing | Navigation structure validation |

### Quantitative Methods
| Method | Best For |
|--------|----------|
| Surveys | Broad patterns, preferences |
| A/B Testing | Design alternatives |
| Analytics | Behavior patterns, funnels |
| Heatmaps | Attention, click patterns |
| Eye Tracking | Visual attention flow |

## Research Process

### 1. Planning
- Define research questions
- Identify user segments
- Select methodology
- Plan timeline and resources
- Set success criteria

### 2. Recruitment
- Define screening criteria
- Recruit participants
- Schedule sessions
- Prepare consent forms
- Set up incentives

### 3. Execution
- Conduct sessions
- Collect data systematically
- Record observations
- Note unexpected findings
- Remain objective

### 4. Analysis
- Transcribe and code data
- Identify patterns and themes
- Triangulate findings
- Generate insights
- Create recommendations

### 5. Reporting
- Executive summary
- Key findings
- Supporting evidence
- Actionable recommendations
- Presentation to stakeholders

## Usability Testing Protocol

### Test Planning
- Define test objectives
- Create task scenarios
- Prepare prototype/product
- Set up recording
- Create observation guide

### Task Design
- Realistic scenarios
- Clear task wording
- Appropriate difficulty
- Cover key flows
- Include edge cases

### During Testing
- Welcome and consent
- Think-aloud protocol
- Minimal intervention
- Note pain points
- Time task completion
- Ask follow-up questions

### Post-Test
- Debrief participant
- Thank and incentivize
- Document observations
- Identify patterns across sessions

## User Interview Guide

### Structure
1. Introduction and consent
2. Background questions
3. Topic exploration
4. Deep dive on key areas
5. Wrap-up and thanks

### Question Types
| Type | Use | Example |
|------|-----|---------|
| Open-ended | Exploration | "Tell me about..." |
| Follow-up | Depth | "Can you elaborate on that?" |
| Clarifying | Understanding | "What did you mean by..." |
| Specific | Details | "How often do you..." |
| Closing | Summary | "Is there anything else?" |

### Best Practices
- Avoid leading questions
- Don't assume or interpret
- Let user tell their story
- Note non-verbal cues
- Stay neutral

## Persona Development

### Persona Components
- Name and photo
- Demographics
- Goals and motivations
- Pain points and frustrations
- Behaviors and habits
- Technology comfort
- Quote that captures essence

### Persona Types
| Type | Purpose |
|------|---------|
| Proto-persona | Quick assumption-based |
| Data-backed | Research-validated |
| Negative | Who you're NOT designing for |

## Journey Mapping

### Components
- User stages/phases
- Touchpoints per stage
- User actions
- Thoughts and feelings
- Pain points
- Opportunities

### Mapping Process
1. Define scope and persona
2. Identify stages
3. Map touchpoints
4. Add emotional layer
5. Identify pain points
6. Generate opportunities

## A/B Testing

### Test Design
- Clear hypothesis
- One variable at a time
- Sufficient sample size
- Statistical significance target
- Duration planning

### Metrics
| Type | Examples |
|------|----------|
| Primary | Conversion, completion |
| Secondary | Engagement, time |
| Guardrail | Bounce, errors |

### Analysis
- Statistical significance check
- Segment analysis
- Secondary metric impact
- Practical significance
- Recommendation

## Accessibility Research

### Testing Areas
- Screen reader compatibility
- Keyboard navigation
- Color contrast
- Touch target sizing
- Cognitive load
- Motion sensitivity

### Methods
- Automated testing (axe, WAVE)
- Manual testing
- User testing with assistive tech
- Expert review

## Research Quality Checklist

- [ ] Sample size adequate
- [ ] Bias minimized
- [ ] Insights actionable
- [ ] Data triangulated
- [ ] Findings validated
- [ ] Recommendations clear
- [ ] Impact measurable
- [ ] Stakeholders aligned

## Research Integration

Research outputs feed into:
- **Product Agent**: Priorities and requirements
- **UX Agent**: User flows and wireframes
- **UI Agent**: Visual design decisions
- **Design Reviewer**: Validation criteria

## Ethics & Consent

- Informed consent before sessions
- Clear purpose explanation
- Privacy and data protection
- Right to withdraw
- Appropriate incentives
- No deceptive practices
